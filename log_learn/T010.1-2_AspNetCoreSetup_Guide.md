# ASP.NET Core Web API Setup - Learning Guide

## Overview
How to create an ASP.NET Core Web API project with specific platform requirements.

## Key Concepts

### 1. Project SDK Types
```xml
<!-- Web API -->
<Project Sdk="Microsoft.NET.Sdk.Web">

<!-- Class Library -->
<Project Sdk="Microsoft.NET.Sdk">

<!-- Worker Service -->
<Project Sdk="Microsoft.NET.Sdk.Worker">
```

### 2. Platform Targeting
For 32-bit COM interop (like ContPAQi SDK):
```xml
<PropertyGroup>
  <TargetFramework>net6.0</TargetFramework>
  <PlatformTarget>x86</PlatformTarget>
</PropertyGroup>
```

**Platform Options:**
| Value | Description |
|-------|-------------|
| AnyCPU | Default, runs as 64-bit on 64-bit OS |
| x86 | Always 32-bit |
| x64 | Always 64-bit |

### 3. Solution Structure
```
SolutionName/
├── SolutionName.sln
├── src/
│   └── ProjectName/
│       ├── ProjectName.csproj
│       ├── Program.cs
│       ├── Controllers/
│       ├── Models/
│       └── Services/
└── tests/
    └── ProjectName.Tests/
        └── ProjectName.Tests.csproj
```

### 4. Creating Projects

**Via CLI:**
```bash
# Create solution
dotnet new sln -n ContpaqiBridge

# Create web api
dotnet new webapi -n ContpaqiBridge -o src/ContpaqiBridge

# Create test project
dotnet new xunit -n ContpaqiBridge.Tests -o tests/ContpaqiBridge.Tests

# Add to solution
dotnet sln add src/ContpaqiBridge
dotnet sln add tests/ContpaqiBridge.Tests
```

### 5. Nullable and Implicit Usings
```xml
<PropertyGroup>
  <Nullable>enable</Nullable>       <!-- Null safety -->
  <ImplicitUsings>enable</ImplicitUsings>  <!-- Less boilerplate -->
</PropertyGroup>
```

## Best Practices

1. **Separate src/ and tests/** - Clear organization
2. **Use .NET LTS versions** - 6.0 or 8.0 for stability
3. **Enable nullable** - Catch null errors at compile time
4. **Match platform to dependencies** - x86 for 32-bit COM
5. **Use solution files** - Even for single projects

## Key Takeaways

1. SDK type determines project capabilities
2. PlatformTarget must match native dependencies
3. Solution structure aids maintainability
4. CLI or IDE can create projects
5. Modern settings reduce boilerplate
